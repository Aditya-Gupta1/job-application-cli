Commands#

Track Job Applications support basic commands to add, delete or update the job applications. Apart from it, it also has the ability to generate a basic report which gives some insights on the applications saved.

Note: By default, all the commands are executed on the entire data. In order to run it on the subset of applications, --start-date and --end-date can be given in YYYY-MM-DD format to perform the operations for application that have the applied_at date between this range (inclusive).

All the commands mentioned henceforth have a --help flag that can be used to get information about them.

Basic Commands#

-v/--version#

Displays application version.

Parameters:

Name Type Description Default
version Optional[bool]

-v or --version flag

 typer.Option(None, '--version', '-v', help="Show the application's version and exit.", callback=_version_callback, is_eager=True)

Examples:

>>>track-job --version
>>>track-job -v

add#

Adds a job application.

Parameters:

Name Type Description Default
company str

Company name applied to.

 typer.Argument(Ellipsis, help='Company applied to')
position str

Job Post applied to.

 typer.Argument(Ellipsis, help='Position applied for')
applied_at str

Date on which the application was submitted for the given job posting in YYYY-MM-DD format. (default: current date)

 typer.Option(datetime.date(datetime.now()).isoformat(), '--applied-at', '-a', help='Date applied at [YYYY-MM-DD]')
status str

Application status. For all possible values, check Status. (default: APPLIED)

 typer.Option('APPLIED', '--status', '-s', help='Current status of the application')

Examples:

>>>track-job add Company1 Position1
>>>track-job add Company1 Position2 --applied-at 2022-11-10 --status tech
>>>track-job add Company1 Position2 -a 2022-11-10 -s offer

ls#

Lists all the job applications present in the database. (or within a specified date range, if given)

Parameters:

Name Type Description Default
start_date Optional[str]

Start date to display applications (default: None)

 typer.Option(None, '--start-date', '-s', help='Start date in YYYY-MM-DD')
end_date Optional[str]

End date to display applications (default: None)

 typer.Option(None, '--end-date', '-e', help='End date in YYYY-MM-DD')

Examples:

>>>track-job ls //displays all the applications
>>>track-job ls -s 2022-10-12 -e 2022-11-12 //displays all the applications from 2022-10-12 to 2022-11-12
Notes

In case the start and the end dates are given, the applications are filtered using the applied_at field.

rm#

Deletes the job application with the given id.

Parameters:

Name Type Description Default
application_id int

ID of an application to be deleted.

 typer.Argument(Ellipsis, help='ID for job application you want to delete')

Examples:

>>>track-job rm 1 //deletes application having ID 1
Notes

The application id can be found using the ls command.

Report Commands#

report#

Generate a report for all the applications (or in the given date range). This is called implicitly when no sub-command is provided to the report command.

Parameters:

Name Type Description Default
ctx typer.Context

Typer context. Provided by typer implicitly.

 required
start_date Optional[str]

Start date to filter applications. (default: None)

 typer.Option(None, '--start-date', '-s', help='start date')
end_date Optional[str]

End date to filter applications. (default: None)

 typer.Option(None, '--end-date', '-e', help='end date')

Examples:

>>>track-job report //generates report for all the applications
>>>track-job report -s 2022-10-12 -e 2022-11-12 //for the given date range

counts#

Get the total number of applications, or within a given date range.

Parameters:

Name Type Description Default
start_date Optional[str]

Start date to count applications. (default: None)

 typer.Option(None, '--start-date', '-s', help='Start counting the applications from this date')
end_date Optional[str]

End date to count applications. (default: None)

 typer.Option(None, '--end-date', '-e', help='Count the applications made till this date')

Examples:

>>>track-job report counts //count of all the applications present
>>>track-job report counts -s 2022-10-12 -e 2022-11-12

status#

Get the counts of application for the given status in the given time range. By default, it'll count all the applications with the given status.

Parameters:

Name Type Description Default
status str

The Status for which to calculate the application counts. See Status for possible valyes.

 typer.Argument(Ellipsis, help='Status of the application')
start_date Optional[str]

Start date to count applications. (default: None)

 typer.Option(None, '--start-date', '-s', help='Start counting the applications from this date')
end_date Optional[str]

End date to count applications. (default: None)

 typer.Option(None, '--end-date', '-e', help='Count the applications made till this date')

Examples:

>>>track-job report status applied
>>>track-job report status tech-interview -s 2022-10-12 -e 2022-11-12

shortlisted#

Gets the number of shortlisted applications within a given time range.

Parameters:

Name Type Description Default
start_date Optional[str]

Start date to count applications. (default: None)

 typer.Option(None, '--start-date', '-s', help='Start counting the applications from this date')
end_date Optional[str]

End date to count applications. (default: None)

 typer.Option(None, '--end-date', '-e', help='Count the applications made till this date.')

Examples:

>>>track-job report shortlisted
>>>track-job report shortlisted -s 2022-10-12 -e 2022-11-12
Notes

Any application which has a status other than Applied and Rejected will be considered as shortlisted.
